一次性設定 / SETUP

// ROTATE ADMIN CREDENTIALS

重跑本頁會覆蓋舊帳密。每次重跑都必須用一把新產生的 PAT， 完成後到 GitHub 把舊 PAT 撤銷（Revoke）— 只改密碼、不換 PAT ＝ 沒有換鎖。
絕對不要使用曾在任何聊天、訊息、Email 出現過的密碼。

STEP 1 — 建立 GitHub 金鑰（PAT）

  1. 開 github.com/settings/personal-access-tokens/new（Fine-grained token）
  2. Token name 隨意（例 ludens-admin）；Expiration 選 1 年
  3. Repository access → Only select repositories → 只勾 ludens-card
  4. Permissions → Repository permissions → Contents → Read and write（其他全部不動）
  5. Generate token → 複製整串貼到下面

STEP 2 — 設定帳密並加密

「產生隨機密碼」用的是瀏覽器密碼學亂數（約 117 bits）。產生後立刻抄進密碼管理器 — 忘了密碼只能重跑本頁換新 PAT。

完成 ✓

  1. 前往 admin.html 用剛設的帳密登入
  2. 如果這不是第一次設定：到 PAT 管理頁 把舊的 token Revoke
